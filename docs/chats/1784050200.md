# Date

2026-07-14

# Context

Cycle V2 is net-new and temporary formats from its development were being
treated as legacy compatibility inputs in new curve-node code.

# Decisions

- Do not preserve intermediate Cycle V2 scaffolding as a long-term migration
  contract.
- Port checked-in presets, fixtures, and tests to the intended typed format.
- Remove compatibility paths once repository-owned inputs have been ported.

# Generalized Rules

- Be deliberately strict about deleting weak or transitional Cycle V2 code.
- Add migration only for an explicit external saved-format compatibility
  requirement, not merely because an earlier development branch used another
  representation.

# Open Questions

- Decide when Cycle V2 file-format compatibility becomes a public product
  contract and document that version boundary explicitly.
