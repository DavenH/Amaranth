# Session Summary

Date: 2026-03-14

## Context

This session reviewed the `lib/src/Curve/V2/` rasterization refactor layout,
especially whether `Runtime` was still an accurate folder name once it held
shared types, workspace storage, orchestration code, sampling helpers, and
concrete rasterizers.

## Decisions

- `Runtime` is too broad for this subsystem and should be split by role.
- Use `Orchestration` for the coordinating layer that wires controls and staged
  execution together.
- Put shared render-domain types and workspace storage in `Core`.
- Keep stage contracts and context types with `Stages`.
- Keep concrete products in `Rasterizers` and waveform sampling helpers in
  `Sampling`.

## Generalized Rules

- When a folder becomes a mixed bucket, rename and split it by architectural
  role rather than preserving a vague label for continuity.
- Treat workspace/stateful storage as core domain support, not orchestration.
- Stage context types should live with stage contracts unless there is a clear
  reusable domain model that stands on its own.
