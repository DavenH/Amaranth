# Date

2026-07-19

# Context

Cycle V2 editor lag exposed that invalidation policies were tested without
governing production work, and that one semantic pointer edit could trigger
duplicated rasterization, serialization, or downstream traversal.

# Decisions

- Give each semantic movement edit a causal identity and make its path through
  the update graph auditable.
- Enforce exactly-once work per edit, node, and derived product, including at
  converging graph paths.
- Keep immediate local editor feedback separate from downstream traversal and
  durable graph publication.
- Let users choose whether signal probes update on gesture release or converge
  continuously on the latest movement edit.
- Keep local editor products synchronous and minimal while downstream preview
  work runs asynchronously from immutable snapshots.
- Cancel or invalidate obsolete same-source downstream work so stale
  calculations cannot delay or overwrite the latest edit result.
- Store Cycle V2 graph documents as canonical JSON, completing the existing
  `Savable`/mesh migration rather than nesting JSON inside XML.
- Represent aggregate mesh and curve models as structured typed node state,
  separate from scalar parameters and editor intent.
- Convert Cycle V2 presets directly to the new format without an XML migration
  layer.
- Gate edit identity and invalidation on canonical effective-value changes,
  after snapping, rounding, and model constraints.
- Make discrete UI controls use the same normalizer as DSP so their visible
  snapping accurately communicates update boundaries.
- Treat Cycle v1's dirty-update graph as the authoritative behavioral model,
  while adapting its guarantees to Cycle V2's dynamic compiled graph rather
  than forcing the legacy raw-pointer graph into the new architecture.

# Generalized Rules

- An invalidation policy is meaningful only when production execution consumes
  it and operation counts can prove the result.
- Assign one identity to a semantic multidimensional edit; do not model its
  coordinates as independent publications.
- Make skipped, deferred, superseded, and executed work distinguishable in
  diagnostics.
- Trace superseded edits explicitly and publish an asynchronous result only if
  its source generation is still current.
- Never encode a domain model as a JSON string inside another persistence
  format; embed its structured JSON value and reuse its authoritative codec.
- A no-op input attempt must not advance revisions, cancel useful pending work,
  restart smoothing, or trigger local/downstream derived computation.
