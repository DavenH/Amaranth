# Date

2026-06-05

# Context

Cycle v2 Trilinear Mesh popup work reached a point where the panel areas were
behaving as real JUCE child components, but the morph/vertex control area and
visual update routing were still temporary. The user asked whether Cycle 1 leaf
widgets such as `CubeDisplay` and `HSlider` could be reused, and whether Cycle
2 should have an updater/update graph like Cycle 1.

# Decisions

- Use normal JUCE child components in Cycle v2 node popups where they solve
  focus, hit testing, cursor, and slider behavior directly.
- Copy or adapt Cycle 1 leaf widgets when their visual/interaction language is
  right, but bind them to explicit Cycle 2 node-local state and callbacks.
- Treat `SingletonRepo` as reusable Amaranth infrastructure, not as a reason to
  pull Cycle 1 global scratch-channel, console-message, settings, or fixed
  update-source assumptions into Cycle v2 node controls.
- Add a Cycle 2 update graph rather than continuing to grow bridge-local manual
  repaint and invalidation rules.
- Split Trimesh rendering into node model, render profiles, 3D surface
  renderer, 2D slice renderer, controls component, and expanded editor shell.

# Generalized Rules

- When porting Cycle 1 UI leaves into Cycle 2, preserve the interaction
  language but make dependencies explicit and node-local.
- Prefer update-graph invalidation for coupled visual/audio artifacts: mesh
  content, morph position, primary axis, 2D slice, 3D grid, intercepts/rails,
  compact preview, expanded panel textures, selected controls, and DSP prep.
- Avoid solving Cycle v2 popup/control problems with broad hand-painted hit
  regions once real components can provide natural JUCE semantics.
- Domain-specific rendering should be profile-driven. Time, spectral magnitude,
  and spectral phase views need distinct backgrounds, scales, gradients, alpha
  behavior, and overlays.

# Open Questions

- Whether Trimesh render profiles should start inside the Trimesh module or be
  introduced immediately as generic waveform/spectrum panel profiles for spy
  nodes and future source nodes.
