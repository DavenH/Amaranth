# Date

2026-07-22

# Context

A successful repository-specific design workflow was promoted into a reusable
skill for contributors and coding agents.

# Decisions

- Store shared project skills under `.agents/skills` rather than a personal
  Codex directory.
- Keep repository skills generic and portable because the repository is public.
- Preserve specialized workflows as concise skills when repeated general
  prompting has produced materially weaker results.

# Generalized Rules

- Put skills intended for all contributors in `.agents/skills`.
- Avoid personal filesystem paths, private assumptions, or product-specific
  metadata unless the skill is deliberately scoped to that product.
- Capture the repeatable reasoning and validation loop, not a transcript or a
  brittle description of one finished artifact.
