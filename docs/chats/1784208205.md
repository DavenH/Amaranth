# Date

2026-07-16

# Context

A complexity review initially treated E3's per-column envelope rasterization as
waste. The user clarified that every E3 column is intentionally a distinct
morph cross-section, while signal traversal columns share one prepared envelope
and advance through its time axis.

# Decisions

- Preserve one cross-section render per E3 morph column; the varying morph
  input makes that cost inherent.
- Rename generic operations such as `renderMesh` to describe the domain result,
  such as rendering a cross-section or slice.
- Pass values that genuinely vary per call explicitly rather than requiring
  callers to mutate ambient member state.
- Keep stable collaborators and saved references as members when they define
  object identity and do not vary per operation.

# Generalized Rules

- Establish which inputs change before classifying repeated work as wasteful.
- Prefer pure-function-shaped method signatures that reveal varying
  dependencies, without creating long parameter lists from stable object state.
- Use domain-specific names that make operation meaning and cost apparent.
