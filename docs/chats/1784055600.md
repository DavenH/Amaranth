# Date

2026-07-14

# Context

Cycle V2 accumulated a large compatibility bridge that copied mature Cycle V1
panel and interaction behavior despite repository reuse rules.

# Decisions

- Treat mature Cycle V1 behavior as authoritative and extract shared cores for
  Cycle V1 and Cycle V2 rather than reimplementing it in adapters.
- Require architectural stop rules, provenance, deletion targets, complexity
  expectations, and negative acceptance criteria in TDD work.
- Prefer semantic interaction tests over numerous low-signal unit tests.

# Generalized Rules

- Do not accept a large compatibility object merely because tests pass.
- Ordinary single-object edits should retain identity and expected O(1)
  behavior; whole-document work belongs at explicit commit boundaries.
- Mark designs in progress while their central god object or duplication
  remains.

# Open Questions

- Determine how much existing low-signal Cycle V2 test coverage should be
  replaced with behavioral and architecture-level tests.
