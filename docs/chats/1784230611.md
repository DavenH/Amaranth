# Date

2026-07-16

# Context

Repository instructions were being observed during initial implementation but
not consistently re-applied as explicit post-implementation review gates. This
allowed working behavior to retain abstraction leaks and compressed style.

# Decisions

- Make the default engineering loop: technical design, technical
  implementation, refactor pass, style check, semantic verification, commit.
- Treat each stage as an active task with evidence rather than passive guidance.
- Put the loop in repository instructions so it runs automatically; do not
  require the user to invoke several skills after every implementation.
- Keep realtime control smoothing in per-voice audio state. Keep immutable
  authoring configuration and graphic requests stateless.
- Extract low-level publication mechanics from high-level orchestration after
  first-pass behavior is established.

# Generalized Rules

- Re-read the production diff after tests pass and actively search for mixed
  abstraction levels, duplication, and leaked concurrency/storage mechanics.
- Passing tests ends the implementation stage, not the engineering loop.
- Use skills for specialized knowledge or tooling, not for mandatory baseline
  engineering discipline.
