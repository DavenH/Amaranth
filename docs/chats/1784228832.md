# Date

2026-07-16

# Context

Cycle v2 needed a clear source and meaning for live Trimesh and Envelope morph
positions while introducing dynamic Envelope preparation.

# Decisions

- Keep modulation entirely in the node graph; do not add a dedicated
  modulation-matrix UI or hidden routing subsystem.
- Treat connected morph signals as absolute effective positions in `[0, 1]`,
  not offsets from persistent node parameters.
- Expose yellow/red/blue control inputs on Trimesh nodes and red/blue control
  inputs on Envelope nodes.
- Let unconnected axes independently fall back to their persisted node values.
- Put depth, polarity, scaling, and signal combination in upstream control
  nodes.

# Generalized Rules

- Prefer explicit typed graph ports over side-channel modulation APIs.
- A runtime control input must not mutate persistent graph authoring state.
- Make fallback semantics per input axis so partially connected controls remain
  predictable.
